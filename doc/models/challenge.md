
# Challenge

*This model accepts additional fields of type Object.*

## Structure

`Challenge`

## Fields

| Name | Type | Tags | Description |
|  --- | --- | --- | --- |
| `sid` | `String` | Optional | A 34 character string that uniquely identifies this Challenge.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^YC[0-9a-fA-F]{32}$` |
| `account_sid` | `String` | Optional | The unique SID identifier of the Account.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^AC[0-9a-fA-F]{32}$` |
| `service_sid` | `String` | Optional | The unique SID identifier of the Service.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^VA[0-9a-fA-F]{32}$` |
| `entity_sid` | `String` | Optional | The unique SID identifier of the Entity.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^YE[0-9a-fA-F]{32}$` |
| `identity` | `String` | Optional | Customer unique identity for the Entity owner of the Challenge. This identifier should be immutable, not PII, length between 8 and 64 characters, and generated by your external system, such as your user's UUID, GUID, or SID. It can only contain dash (-) separated alphanumeric characters. |
| `factor_sid` | `String` | Optional | The unique SID identifier of the Factor.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^YF[0-9a-fA-F]{32}$` |
| `date_created` | `DateTime` | Optional | The date that this Challenge was created, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. |
| `date_updated` | `DateTime` | Optional | The date that this Challenge was updated, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. |
| `date_responded` | `DateTime` | Optional | The date that this Challenge was responded, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. |
| `expiration_date` | `DateTime` | Optional | The date-time when this Challenge expires, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. The default value is five (5) minutes after Challenge creation. The max value is sixty (60) minutes after creation. |
| `status` | [`ChallengeEnumChallengeStatuses`](../../doc/models/challenge-enum-challenge-statuses.md) | Optional | The Status of this Challenge. One of `pending`, `expired`, `approved` or `denied`. |
| `responded_reason` | [`ChallengeEnumChallengeReasons`](../../doc/models/challenge-enum-challenge-reasons.md) | Optional | Reason for the Challenge to be in certain `status`. One of `none`, `not_needed` or `not_requested`. |
| `details` | `Object` | Optional | Details provided to give context about the Challenge. Intended to be shown to the end user. |
| `hidden_details` | `Object` | Optional | Details provided to give context about the Challenge. Intended to be hidden from the end user. It must be a stringified JSON with only strings values eg. `{"ip": "172.168.1.234"}` |
| `metadata` | `Object` | Optional | Custom metadata associated with the challenge. This is added by the Device/SDK directly to allow for the inclusion of device information. It must be a stringified JSON with only strings values eg. `{"os": "Android"}`. Can be up to 1024 characters in length. |
| `factor_type` | [`ChallengeEnumFactorTypes`](../../doc/models/challenge-enum-factor-types.md) | Optional | The Factor Type of this Challenge. Currently `push` and `totp` are supported. |
| `url` | `String` | Optional | The URL of this resource. |
| `links` | `Object` | Optional | Contains a dictionary of URL links to nested resources of this Challenge. |
| `additional_properties` | `Hash[String, Object]` | Optional | - |
