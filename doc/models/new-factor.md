
# New Factor

*This model accepts additional fields of type Object.*

## Structure

`NewFactor`

## Fields

| Name | Type | Tags | Description |
|  --- | --- | --- | --- |
| `sid` | `String` | Optional | A 34 character string that uniquely identifies this Factor.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^YF[0-9a-fA-F]{32}$` |
| `account_sid` | `String` | Optional | The unique SID identifier of the Account.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^AC[0-9a-fA-F]{32}$` |
| `service_sid` | `String` | Optional | The unique SID identifier of the Service.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^VA[0-9a-fA-F]{32}$` |
| `entity_sid` | `String` | Optional | The unique SID identifier of the Entity.<br><br>**Constraints**: *Minimum Length*: `34`, *Maximum Length*: `34`, *Pattern*: `^YE[0-9a-fA-F]{32}$` |
| `identity` | `String` | Optional | Customer unique identity for the Entity owner of the Factor. This identifier should be immutable, not PII, length between 8 and 64 characters, and generated by your external system, such as your user's UUID, GUID, or SID. It can only contain dash (-) separated alphanumeric characters. |
| `binding` | `Object` | Optional | Contains the `factor_type` specific secret and metadata.<br>For push, this is `binding.public_key` and `binding.alg`.<br>For totp, this is `binding.secret` and `binding.uri`. The `binding.uri` property is generated following the [google authenticator key URI format](https://github.com/google/google-authenticator/wiki/Key-Uri-Format), and `Factor.friendly_name` is used for the “accountname” value and `Service.friendly_name` or `Service.totp.issuer` is used for the `issuer` value.<br><br>The Binding property is ONLY returned upon Factor creation. |
| `options` | `Object` | Optional | - |
| `date_created` | `DateTime` | Optional | The date that this Factor was created, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. |
| `date_updated` | `DateTime` | Optional | The date that this Factor was updated, given in [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format. |
| `friendly_name` | `String` | Optional | The friendly name of this Factor. This can be any string up to 64 characters, meant for humans to distinguish between Factors.<br>For `factor_type` `push`, this could be a device name.<br>For `factor_type` `totp`, this value is used as the “account name” in constructing the `binding.uri` property.<br>At the same time, we recommend avoiding providing PII. |
| `status` | [`NewFactorEnumFactorStatuses`](../../doc/models/new-factor-enum-factor-statuses.md) | Optional | The Status of this Factor. One of `unverified` or `verified`. |
| `factor_type` | [`NewFactorEnumFactorTypes`](../../doc/models/new-factor-enum-factor-types.md) | Optional | The Type of this Factor. Currently `push` and `totp` are supported. |
| `config` | `Object` | Optional | An object that contains configurations specific to a `factor_type`. |
| `metadata` | `Object` | Optional | Custom metadata associated with the factor. This is added by the Device/SDK directly to allow for the inclusion of device information. It must be a stringified JSON with only strings values eg. `{"os": "Android"}`. Can be up to 1024 characters in length. |
| `url` | `String` | Optional | The URL of this resource. |
| `additional_properties` | `Hash[String, Object]` | Optional | - |
